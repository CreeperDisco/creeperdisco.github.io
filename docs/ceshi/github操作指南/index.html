基础概念#

1. 什么是Git和GitHub#

Github和Git的关系

关于 GitHub#

GitHub 是一种基于云的平台，可在其中存储、共享并与他人一起编写代码。

通过将代码存储在 GitHub 上的“存储库”中，你可以：

  • “展示或共享”你的工作。
  • 持续“跟踪和管理”对代码的更改。
  • 让其他人“审查”你的代码，并提出改进建议。
  • 在共享的项目中开展“协作”，无需担心这些更改会在准备好集成更改之前影响协作者的工作。

协作式工作是 GitHub 最基本的功能之一，该功能由开源软件 Git 实现，而 GitHub 是以该软件为基础进行构建的。

关于 Git#

Git 是一种可以智能追踪文件中的更改的版本控制系统。 在你和一组人员同时对同一文件进行更改时，Git 的价值就体现出来了。

通常，要在基于 Git 的工作流中执行此操作，需要：

  • 为你（和协作者）正在处理的文件的主副本“创建分支”。
  • 在自己的个人分支上独立安全地对文件“进行编辑”。
  • 让 Git 智能地将特定更改“合并”回文件的主副本，从而确保更改不会影响其他人的更新。
  • 让 Git “跟踪”你和其他人的更改，这样就可以一直使用项目的最新版本。

若要自行尝试使用 Git，请参阅 开始使用 Git。

—————————————————————————————————————————–

.gitignore 文件是 Git 版本控制系统中的一个重要组件，它允许开发者指定在项目提交过程中应该被 Git 忽略的文件或目录。这个文件通常位于 Git 仓库的根目录下，但也可以在仓库的任何子目录中创建多个 .gitignore 文件。

.gitignore 文件的基本规则

  • 斜杠 / 开头 表示规则适用于根目录。

  • 斜杠 / 结尾 表示忽略的是文件夹。

  • 星号 * 用作通配符，匹配任意数量的字符。

  • 问号 ? 匹配单个字符。

  • 方括号 [] 匹配方括号内的任意字符。

  • 感叹号 ! 用于否定模式，即不忽略匹配的文件。

  • 井号 # 用于添加注释。

例如，要忽略根目录下的 bin 文件夹，可以在 .gitignore 文件中添加以下规则：

/bin/

而要忽略所有 .log 文件，无论它们位于哪个目录下，可以使用：

*.log

忽略文件夹但包含特定文件

如果你想忽略一个文件夹中的大部分内容，但想保留特定的文件或子目录，你可以使用 ! 来重新包含这些项目。例如，忽略 bin 文件夹但保留 run.sh 脚本：

/bin/

!/bin/run.sh —————————————————————————————————————————–

然后你就可以放心把代码扔进目录了，不会这个也要我教吧 。#
